# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build Commands

```bash
# Configure (from project root, using existing build/ dir or a new one)
cmake -B build -S .

# Build
cmake --build build

# Run all tests
cd build && ctest

# Run a single test by name pattern
ctest -R tjunittest-shared
ctest -R cjpeg-shared-rgb-islow

# Build with specific options
cmake -B build -S . -DWITH_SIMD=0 -DWITH_TURBOJPEG=0 -DWITH_TESTS=1
```

Key CMake options (defaults shown): `WITH_SIMD=ON`, `WITH_TURBOJPEG=ON`, `WITH_TESTS=ON`, `WITH_TOOLS=ON`, `WITH_ARITH_DEC=ON`, `WITH_ARITH_ENC=ON`, `WITH_JPEG7=OFF`, `WITH_JPEG8=OFF`, `REQUIRE_SIMD=OFF`, `WITH_FUZZ=OFF`.

## Architecture

**Dual API:** The project exposes two APIs built from the same core:
- **libjpeg API** (`jpeglib.h`, `libjpeg.so.62`) — Traditional IJG low-level scanline API. ABI-compatible with libjpeg v6b by default; v7/v8 emulation via CMake flags.
- **TurboJPEG API** (`turbojpeg.h`, `libturbojpeg.so.0`) — Higher-level whole-image API wrapping libjpeg. Handles entire images in memory, return-code error handling, built-in YUV/transform support.

**Multi-precision via compile-time wrappers:** Source files like `jccolor.c`, `jdsample.c` use precision-independent tokens (e.g., `_JSAMPLE`). The header `src/jsamplecomp.h` maps these to concrete types based on `BITS_IN_JSAMPLE`. Wrapper files in `src/wrapper/` (e.g., `jccolor-8.c`, `jccolor-12.c`, `jccolor-16.c`) set the precision macro then `#include` the core file. These wrappers are auto-generated by `cmakescripts/GenerateWrappers.cmake` at configure time.

**SIMD acceleration** (`simd/`): Runtime-dispatched via function pointers. `simd/jsimd.c` detects CPU features at init and plugs the best available SIMD implementation (SSE2/AVX2 on x86, NEON on ARM, AltiVec on PowerPC, MMI on MIPS64, RVV on RISC-V). Override with env var `JSIMD_FORCENONE=1`. SIMD coverage is 8-bit lossy JPEG only.

**No `add_subdirectory()`:** The top-level CMakeLists.txt explicitly prohibits being used as a subdirectory. Downstream projects must use `ExternalProject_Add()`.

## Coding Conventions

- K&R indentation style, 2 spaces per level, no tabs
- Lowercase with underscores for names (`multi_word_name`); uppercase for preprocessor/enum (`MULTI_WORD_NAME`)
- Function definitions use `GLOBAL(type)`, `LOCAL(type)`, or `METHODDEF(type)` macros
- External declarations use `EXTERN` macro
- Library source files begin with `j` prefix; non-library files (cjpeg, djpeg) do not
- Compression and decompression code live in separate source files
- No global or static variables (except read-only constant tables); use the common data structures
- Avoid direct file I/O, malloc, or error printouts; use the library's provided routines
- Use [checkstyle](https://github.com/libjpeg-turbo/checkstyle) to validate formatting

## Key Source Layout

- `src/` — Core library: compression (`jc*.c`), decompression (`jd*.c`), DCT (`jf*.c`, `ji*.c`), infrastructure (`jmemmgr.c`, `jerror.c`, `jcomapi.c`), image I/O (`rd*.c`, `wr*.c`), tools (`cjpeg.c`, `djpeg.c`, `jpegtran.c`), TurboJPEG impl (`turbojpeg.c`, `turbojpeg-mp.c`)
- `src/wrapper/` — Auto-generated precision-specific wrappers (do not edit directly; modify the template source and regenerate)
- `simd/` — SIMD modules per architecture: `x86_64/` (NASM), `i386/` (NASM), `arm/` (Neon intrinsics), `powerpc/` (AltiVec intrinsics), `mips64/` (MMI intrinsics), `riscv64/` (RVV)
- `sharedlib/` — Shared library build directory (separate for MSVC CRT handling)
- `test/` — Shell script test templates (`.in` files with CMake variable substitution)
- `testimages/` — Test image assets
- `fuzz/` — LibFuzzer targets (C++)
